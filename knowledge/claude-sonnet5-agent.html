2. Agent 核心能力解析

2.1 工具调用（Tool Use）

能力描述: Sonnet 5 能够：

  1. 理解自然语言指令，自动选择所需工具
  2. 多步骤工具链编排（链式调用）
  3. 错误处理与重试（工具调用失败自动调整策略）
  4. 并行工具调用（同时调用多个独立工具）

示例：复杂任务自动拆解

用户指令:

"帮我分析 GitHub 仓库 anthropics/anthropic-quickstarts 的代码质量，生成报告，并发送邮件给团队。"

Sonnet 5 自动执行:

步骤 1: 调用 github_clone_repo 工具 → 克隆仓库
步骤 2: 调用 code_analyzer 工具 → 分析代码质量
  ├─ 静态分析（复杂度、重复代码、测试覆盖率）
  └─ 输出 JSON 报告
步骤 3: 调用 generate_pdf 工具 → 生成 PDF 报告
步骤 4: 调用 send_email 工具 → 发送邮件
  ├─ 收件人: 从通讯录工具查询 "Agent 团队"
  └─ 附件: PDF 报告

关键提升:

  • 上下文理解: 能理解"团队"指代谁（通过通讯录工具）
  • 容错能力: 如果 generate_pdf 失败，自动降级为 Markdown 报告
  • 并行执行: 步骤 2 和步骤 3 可部分并行（分析完成后立即开始生成报告）

2.3 规划与推理（Planning & Reasoning）

混合推理模式（继承自 Sonnet 3.7）

两种模式:

  1. 即时响应模式 (Instant Response)

    • 适用于: 简单问答、代码补全、快速查询
    • 延迟: < 1 秒

  2. 扩展思考模式 (Extended Thinking)

    • 适用于: 复杂推理、多步骤规划、架构设计
    • 特点: 生成完整思维链（Chain of Thought）
    • 延迟: 5-60 秒（取决于复杂度）

启用方式:

from anthropic import Anthropic

client = Anthropic()

# 即时响应模式（默认）
response = client.messages.create(
    model="claude-sonnet-5",
    messages=[{"role": "user", "content": "什么是 RAG？"}]
)

# 扩展思考模式
response = client.messages.create(
    model="claude-sonnet-5",
    messages=[{"role": "user", "content": "设计一个 Agent 知识库系统"}],
    thinking={
        "type": "extended",
        "budget_tokens": 10000  # 分配 10K tokens 用于思考
    }
)

print(response.thinking)  # 输出思维链
print(response.content)    # 输出最终答案

示例：自动购物

from anthropic import Anthropic

client = Anthropic()

response = client.messages.create(
    model="claude-sonnet-5",
    tools=[
        {
            "name": "computer_use",
            "description": "控制计算机浏览器",
            "input_schema": {
                "type": "object",
                "properties": {
                    "action": {
                        "type": "string",
                        "enum": ["screenshot", "click", "type", "scroll"]
                    },
                    "coordinate": {"type": "array", "items": {"type": "number"}},
                    "text": {"type": "string"}
                }
            }
        }
    ],
    messages=[
        {"role": "user", "content": "帮我在 Amazon 上找到一个价格低于 $50 的 Python 书籍，并加入购物车"}

)

# 1. 打开 Amazon.com（screenshot → 识别搜索框 → click → type "Python books" → press Enter）

3. 多智能体编排（Multi-Agent Orchestration）

发布时间: 2026-05-06（Code w/ Claude 2026 开发者大会）

3.1 架构设计

                    ┌───────────────────┐
                    │   Lead Agent      │
                    │ (Claude Sonnet 5)│
                    └────────┬──────────┘
                             │ 任务拆解
             ┌───────────────┼───────────────┐
             │               │               │
      ┌──────▼──────┐ ┌─────▼──────┐ ┌─────▼──────┐
      │ Specialist  │ │ Specialist │ │ Specialist │
      │ Agent 1     │ │ Agent 2    │ │ Agent 3    │
      │ (Coding)    │ │ (Testing)  │ │ (Docs)     │
      └─────────────┘ └────────────┘ └────────────┘
        (Sonnet 5)      (Sonnet 5)      (Sonnet 5)

工作流程:

  1. Lead Agent: 理解用户需求，拆解为子任务
  2. Specialist Agents: 并行执行子任务（最多 20 个）
  3. 结果聚合: Lead Agent 汇总结果，生成最终输出

3.2 实战案例：全栈应用开发

用户指令:

"帮我开发一个 Todo List 应用，包括前端（React）、后端（FastAPI）、数据库（PostgreSQL）、Docker 部署、单元测试"

多智能体编排:

Lead Agent (Sonnet 5)
  ├─ 子任务 1: 前端开发 → Specialist Agent 1 (React)
  ├─ 子任务 2: 后端开发 → Specialist Agent 2 (FastAPI)
  ├─ 子任务 3: 数据库设计 → Specialist Agent 3 (PostgreSQL)
  ├─ 子任务 4: Docker 配置 → Specialist Agent 4
  └─ 子任务 5: 测试编写 → Specialist Agent 5 (Pytest + Jest)

并行执行（最多 5 个 Agent 同时工作）
  → 20 分钟内完成（单 Agent 需要 2 小时）

成本优势:

  • 单 Agent: 2 小时 × $15/小时 = $30
  • 多 Agent: 20 分钟 × 5 Agents × $15/小时 = $25
  • 节省: 时间 83%，成本 17%

6.2 实现步骤

Step 1: 环境准备

pip install anthropic langchain langgraph github3.py pytest

Step 2: 定义工具

from langchain.tools import tool
import subprocess

@tool
def run_terminal_command(command: str) -> str:
    """运行终端命令"""
    result = subprocess.run(command, shell=True, capture_output=True, text=True)
    return result.stdout if result.returncode == 0 else result.stderr

@tool
def read_file(filepath: str) -> str:
    """读取文件内容"""
    with open(filepath, 'r') as f:
        return f.read()

@tool
def write_file(filepath: str, content: str) -> str:
    """写入文件内容"""
    with open(filepath, 'w') as f:
        f.write(content)
    return f"文件 {filepath} 已写入"

@tool
def run_tests() -> str:
    """运行单元测试"""
    result = subprocess.run("pytest", capture_output=True, text=True)
    return result.stdout

Step 3: 构建 Agent

from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain.prompts import ChatPromptTemplate
from langchain_anthropic import ChatAnthropic

# 1. 初始化 LLM
llm = ChatAnthropic(model="claude-sonnet-5", temperature=0)

# 2. 定义 Prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个高级 Python 开发工程师，能够理解代码库、修复 bug、编写测试。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])

# 3. 创建 Agent
tools = [run_terminal_command, read_file, write_file, run_tests]
agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 4. 运行 Agent
executor.invoke({"input": "找出所有的 TODO 注释，并生成一个 TODO.md 文件"})

Step 4: 多智能体编排（高级）

from langgraph.graph import StateGraph, END

# 1. 定义状态
class AgentState(TypedDict):
    messages: list
    task: str
    results: list

# 2. 定义 Lead Agent
def lead_agent(state: AgentState):
    # 拆解任务
    response = llm.invoke(f"拆解以下任务为子任务：{state['task']}")
    subtasks = response.content.split("\n")
    state["results"] = subtasks
    return state

# 3. 定义 Specialist Agent
def specialist_agent(state: AgentState, subtask: str):
    # 执行子任务
    response = llm.invoke(f"执行子任务：{subtask}")
    state["results"].append(response.content)
    return state

# 4. 构建图
workflow = StateGraph(AgentState)
workflow.add_node("lead", lead_agent)
workflow.add_node("specialist", specialist_agent)
workflow.add_edge("lead", "specialist")
workflow.add_edge("specialist", END)

app = workflow.compile()

8.2 常见陷阱

❌ Don't:

  1. 不要过度依赖计算机使用（Computer Use）→ 优先使用 API
  2. 不要忽略错误重试 → 实现指数退避（exponential backoff）
  3. 不要在小上下文模型中传入大量文档 → 使用 RAG 或向量数据库
  4. 不要忘记设置 max_tokens → 避免意外高额账单
