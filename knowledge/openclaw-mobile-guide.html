OpenClaw 移动端应用使用指南

OpenClaw 移动端应用于 **2026 年 6 月 30 日**正式发布 iOS 和 Android 原生客户端，分别上架 **Apple App Store** 和 **Google Play Store**。该应用的定位需要首先澄清一个关键认知：

概述

OpenClaw 移动端应用于 2026 年 6 月 30 日正式发布 iOS 和 Android 原生客户端，分别上架 Apple App Store 和 Google Play Store。该应用的定位需要首先澄清一个关键认知：

它是什么

  • 与你的私有 OpenClaw 网关配对的联动客户端
  • 提供随时随地与本地 AI 助手交互的遥控面板
  • 支持聊天、语音通话、审批管理和设备感知自动化的全能移动入口

它不是什么

  • ❌ 不是独立运行的 AI 应用——所有 AI 推理均在网关端完成，手机仅作为交互终端
  • ❌ 不是云服务——没有 OpenClaw 官方服务器中转，数据流向完全是「手机 → 你的网关 → 你的本地 LLM」
  • ❌ 不在手机本地运行模型——无需手机端 GPU 或 NPU，对设备性能要求极低

架构概览

graph LR
    A[📱 OpenClaw Mobile] -->|加密通道| B[🏠 私有 OpenClaw 网关]
    B --> C[💻 本地 LLM / API]
    B --> D[🔧 Tools & Skills]
    B --> E[📦 Memory & Knowledge]

这种「瘦客户端 + 私有网关」的架构确保了计算在本地、交互在掌中的设计哲学。

安装指南

iOS 用户

  1. 打开 App Store，搜索「OpenClaw」
  2. 确认开发者为 OpenClaw Project，点击下载
  3. 最低系统要求：iOS 17.0 及以上
  4. 首次启动时授予必要的通知和麦克风权限（语音通话需要）

Android 用户

方式一：Google Play Store（推荐）

  1. 打开 Google Play Store，搜索「OpenClaw」
  2. 确认开发者为 OpenClaw Project，点击安装
  3. 最低系统要求：Android 12 (API 31) 及以上

方式二：APK 旁载（适用于无 Google 服务的设备）

  1. 前往 OpenClaw 官方 GitHub Releases 页面
  2. 下载最新版 openclaw-mobile-{version}.apk
  3. 在系统设置中允许「安装未知来源应用」
  4. 点击 APK 文件完成安装
  5. ⚠️ 安全提示：务必核对 APK 的 SHA256 校验值，确保与官方发布一致

安装后第一步

首次打开应用将直接进入配对向导界面，此时需要准备好你的 OpenClaw 网关。

配对流程

配对是将手机客户端与你的私有 OpenClaw 网关建立安全信任关系的关键步骤。支持两种方式：

方式一：QR 码配对（推荐，更便捷）

  1. 在网关所在机器上，打开 OpenClaw 控制台或 Web UI
  2. 进入 Settings → Devices → Pair New Device
  3. 点击 Generate QR Code，屏幕将显示一次性配对 QR 码
  4. 在手机端配对向导中选择「扫码配对」
  5. 将手机摄像头对准屏幕上的 QR 码
  6. 自动完成证书交换和密钥协商，通常 3 秒内完成

注意事项：

  • QR 码有效期为 5 分钟，过期需重新生成
  • 手机和网关需处于同一局域网（或网关已配置公网可达）
  • 网关端需开启 mobile feature flag

方式二：Token 手动输入

适用于远程配对或无法使用摄像头的场景：

  1. 在网关端执行：

    openclaw gateway pair generate --label "My Phone"

  2. 终端会输出一个一次性配对 Token，格式类似：

    oclaw-pair-v1.aBcDeFgHiJkLmNoPqRsTuVwXyZ.1234567890

  3. 在手机端选择「手动输入 Token」

  4. 复制粘贴 Token，点击配对

  5. 确认设备指纹一致后完成配对

Token 有效期：10 分钟，使用后立即失效。

配对后配置

配对成功后进入设备设置界面：

配置项 说明
设备名称 自定义本设备显示名称（如「我的 iPhone 15」）
默认模型 为该移动端选择默认使用的 LLM 模型
通知偏好 审批请求通知、消息推送等
生物识别锁定 启用 Face ID / Touch ID / 指纹保护应用入口

功能详解

1. 聊天界面与模型选择

移动端聊天界面延续 OpenClaw Web UI 的设计语言，核心交互包括：

模型切换器

  • 顶部工具栏显示当前使用的模型名称
  • 点击可展开模型列表，切换任意网关端已配置的模型
  • 支持实时切换，切换后新一轮对话即刻生效
  • 型号标签显示颜色区分：绿色 = 本地模型、蓝色 = API 模型

会话管理

  • 左侧滑出会话列表，支持新建、重命名、归档、删除
  • 会话历史完全存储在网关端，手机端不落盘任何消息内容
  • 支持搜索历史消息（全文检索由网关端提供）

消息增强

  • Markdown 渲染与代码语法高亮
  • 图片/文件预览（文件通过网关中转）
  • 支持长按消息进行复制、重新生成、引用回复
  • 流式输出实时呈现，打字机效果

2. 语音通话（Realtime Voice）

语音通话是移动端的核心亮点功能，实现真正的「边走边聊」体验。

配置路径： Settings → Voice

设置项 选项 说明
STT 引擎 Whisper (本地) / Deepgram / Azure / Apple Speech 语音转文字引擎选择
TTS 引擎 ElevenLabs / Edge TTS / OpenAI TTS / 系统 TTS 文字转语音引擎选择
VAD 灵敏度 低 / 中 / 高 语音活动检测敏感度
打断模式 开启 / 关闭 允许用户说话打断 AI 回复
通话语言 自动检测 / 指定语言 默认自动检测

通话体验：

  • 点击右下角 🎙️ 按钮进入通话界面
  • 支持免提模式和听筒模式切换
  • 通话界面实时显示 VU 音量指示条
  • 后台通话支持（iOS 需开启 Background Modes → Audio）
  • 通话记录同步至网关

性能建议：

  • 如果网关部署在家庭 NAS / 本地 PC，建议使用本地 Whisper 以获得最低延迟
  • 移动网络环境下推荐 Deepgram 或 Azure，对丢包容忍度更高
  • iOS 系统 TTS 离线可用，适合户外无网络场景（需预先下载语音包）

3. 审批管理（Tool Approvals）

当 OpenClaw 执行需要人工审批的工具调用时（如发送邮件、执行命令、文件写入等），审批请求会实时推送到手机端。

审批流程：

sequenceDiagram
    participant G as 网关
    participant M as 手机端
    participant U as 用户
    
    G->>M: 推送审批请求（工具名 + 参数预览）
    M->>M: 本地通知弹出
    U->>M: 查看详情 / 审批 / 拒绝
    M->>G: 审批结果
    G->>G: 继续 / 取消执行

审批界面：

  • 通知栏横幅展示审批摘要（工具名 + 简要描述）
  • 点击进入详情页，完整展示工具调用参数（JSON 格式化，支持折叠）
  • 一键批准 / 一键拒绝，支持长按填写拒绝理由
  • 支持「本次会话自动批准」——针对同一工具的后续调用不再提示（可随时撤销）
  • 审批历史可在 Settings → Approval History 中查看

安全红线提示：

⚠️ auto-approve 仅对当前配对会话有效，断开后重置。不建议为 exec、write 等高危操作启用自动批准。

4. 设备感知自动化（Paired Node）

这是 OpenClaw 移动端的差异化能力——将手机本身变为一个 Node，网关可以感知手机端的环境信息并触发自动化。

作为 Paired Node 提供的能力：

能力 说明 典型场景
📍 位置感知 上报 GPS / 粗略位置 「到公司附近时提醒打卡」
🔔 通知读取 读取手机通知栏内容 「收到验证码自动填入电脑端」
📷 相机调用 拍照并回传网关 「扫描文档 → OCR → 存入知识库」
🎤 远程录音 网关触发手机录音 会议录音辅助
🌐 网络状态 Wi-Fi / 蜂窝 / 信号强度 自动化触发条件

配置路径： Settings → Paired Node

  1. 开启 Node Mode 开关
  2. 逐项授权所需权限（位置、相机、通知、麦克风等）
  3. 设置数据上报策略：
    • 持续上报：实时同步，功耗较高
    • 按需触发：仅当网关主动查询时上报（推荐）
    • 定时上报：按固定间隔（5 / 15 / 30 分钟）

隐私保护：

  • 所有 Node 数据仅在手机与你的私有网关之间传输
  • 可通过 Privacy Dashboard 查看过去 24 小时内哪些数据被读取
  • 支持地理围栏模糊化：将精确 GPS 模糊至 500m 范围

安全与隐私：「本地优先」原则详解

OpenClaw 移动端的安全模型以「本地优先」（Local-First）为核心理念：

数据流向保证

数据类型 流经路径 云端接触
聊天消息 手机 ↔ 你的网关 ❌ 不经过任何云服务
语音数据 手机 → 你的网关 → STT 引擎 ⚠️ 取决于 STT 提供商选择
审批操作 手机 ↔ 你的网关 ❌ 不经过任何云服务
Node 传感器数据 手机 → 你的网关 ❌ 不经过任何云服务

通信安全

  • 手机与网关之间的连接使用 TLS 1.3 + mTLS 双向认证
  • 配对时交换的证书有设备指纹绑定，防止中间人攻击
  • 支持通过 Tailscale / WireGuard 建立 VPN 隧道后通信（远端访问场景）

手机端保护

  • 应用沙盒内不存储任何对话历史明文
  • 支持生物识别锁（Face ID / Touch ID / Android Biometric）保护应用入口
  • 闲置超时自动锁定（可配置 1 / 5 / 15 分钟）
  • 后台截图模糊处理（iOS App Switcher）

网关端安全建议

  1. 网关不要暴露在公网，始终通过 VPN 或 SSH 隧道访问
  2. 定期轮换配对 Token，撤销不再使用的设备
  3. 在网关端为移动设备设置独立权限策略（如禁止某些高危工具的执行）

故障排查

常见问题

Q：配对时提示「未发现网关」

  • 确认手机和网关处于同一局域网
  • 检查网关端防火墙是否放行配对端口（默认 TCP 18789）
  • 尝试在网关端使用 --advertise 参数指定网卡 IP
openclaw gateway start --advertise 192.168.1.100

Q：连接频繁断开

  • 检查移动网络切换（Wi-Fi ↔ 蜂窝）是否导致 IP 变更
  • 在高级设置中开启「自动重连」和「保活心跳」
  • 如果通过公网连接，确认 DDNS 解析是否正常

Q：语音通话延迟高

  • 尝试切换 STT 引擎：本地 Whisper < API 方案
  • 降低音频采样率（设置 → Voice → 音频质量 → 低带宽模式）
  • 检查网关端 GPU 是否被其他任务占用

Q：审批通知收不到

  • iOS：确认「设置 → 通知 → OpenClaw → 允许通知」已开启
  • Android：确认未将 OpenClaw 加入省电优化白名单
  • 确认网关端 notifications 插件已启用

Q：APK 安装后闪退

  • 确认 Android 版本 ≥ 12
  • 尝试清除应用缓存后重试
  • 使用 adb logcat 抓取崩溃日志上报 GitHub Issues

诊断模式

  1. 手机端进入 Settings → About → Diagnostic Mode

  2. 网关端执行：

    openclaw gateway diagnose --device "<设备名称>"

  3. 查看完整的连接质量报告，包括延迟、丢包率、证书状态等
