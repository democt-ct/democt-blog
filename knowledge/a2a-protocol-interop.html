A2A 协议与 Agent 互操作

> **核心目标**: 让不同框架、不同厂商的 Agent 能够互相通信、协作、交换数据 > > **类比**: A2A 是 Agent 的"HTTP 协议"，MCP 是 Agent 的"REST API"

核心目标: 让不同框架、不同厂商的 Agent 能够互相通信、协作、交换数据

类比: A2A 是 Agent 的"HTTP 协议"，MCP 是 Agent 的"REST API"

1. 为什么需要 A2A 协议？

1.1 当前 Agent 生态的碎片化

框架 厂商 通信方式 互操作性
AutoGen Microsoft 内存消息队列 ❌ 仅支持 AutoGen Agent
CrewAI CrewAI 顺序/层次流程 ❌ 仅支持 CrewAI Agent
LangGraph LangChain 状态图 ❌ 仅支持 LangChain Agent
OpenAI Assistants OpenAI REST API ⚠️ 仅支持 OpenAI 生态

问题: 一个用 AutoGen 构建的 Agent 无法直接与一个用 CrewAI 构建的 Agent 协作。

1.2 A2A 协议的价值

  • 解耦: Agent 不需要知道对方的实现细节，只需要遵循协议
  • 互操作: 不同框架、不同语言的 Agent 可以协作
  • 可扩展: 新增 Agent 类型时，现有 Agent 无需修改
  • 标准化: 类似 HTTP 对 Web 的意义，A2A 是 Agent 互联网的基石

2. A2A 协议详解（Agent2Agent Protocol）

2.1 什么是 A2A？

A2A (Agent2Agent Protocol) 是 Google 于 2025 年推出的开放协议，旨在实现 Agent 之间的互操作。

核心概念:

  • Agent Card: Agent 的"名片"，描述其能力、技能、端点
  • Task: Agent 执行的任务单元
  • Message: Agent 之间的通信单元
  • Artifact: 任务产生的输出（文件、数据、消息）

2.2 A2A 架构

sequenceDiagram
    participant Client as Client Agent
    participant Server as Server Agent
    
    Client->>Server: 1. 获取 Agent Card (.well-known/agent.json)
    Server-->>Client: 返回 Agent Card (JSON)
    
    Client->>Server: 2. 创建 Task (POST /tasks)
    Server-->>Client: 返回 Task ID
    
    loop 任务执行
        Client->>Server: 3. 查询 Task 状态 (GET /tasks/{id})
        Server-->>Client: 返回状态 (working/completed/failed)
    end
    
    Client->>Server: 4. 获取结果 (GET /tasks/{id}/artifacts)
    Server-->>Client: 返回 Artifact

2.3 Agent Card 格式

端点: https://agent.example.com/.well-known/agent.json

{
  "name": "Code Reviewer Agent",
  "description": "Reviews Python code for style, bugs, and performance",
  "version": "1.0.0",
  "endpoints": {
    "tasks": "https://agent.example.com/tasks",
    "stream": "https://agent.example.com/stream"
  },
  "skills": [
    {
      "id": "code_review",
      "name": "Code Review",
      "description": "Review code and provide feedback",
      "input_types": ["text/x-python"],
      "output_types": ["application/json"]
    }
  ],
  "authentication": {
    "type": "Bearer",
    "instructions_url": "https://agent.example.com/auth"
  }
}

字段说明:

  • name: Agent 名称
  • description: Agent 功能描述
  • skills: Agent 支持的技能列表
  • endpoints: Task 创建、状态查询、流式响应等端点
  • authentication: 认证方式

2.4 Task 生命周期

stateDiagram-v2
    [*] --> submitted: 创建 Task
    submitted --> working: 开始执行
    working --> input_required: 需要用户输入
    input_required --> working: 用户提供输入
    working --> completed: 执行成功
    working --> failed: 执行失败
    completed --> [*]
    failed --> [*]

Task 状态:

  • submitted: 已提交，等待执行
  • working: 执行中
  • input_required: 需要用户提供额外输入
  • completed: 执行成功
  • failed: 执行失败

Task 创建请求:

    "session_id": "sess_123",

2.6 流式响应（Streaming）

对于长时间任务，A2A 支持 Server-Sent Events (SSE) 流式返回进度。

请求:

GET /stream?task_id=task_789 HTTP/1.1
Accept: text/event-stream

响应:

event: task_status
data: {"status": "working", "progress": 20}

event: task_status
data: {"status": "working", "progress": 50}

event: artifact
data: {"partial": true, "content": {"intermediate_result": "..."}}

event: task_status
data: {"status": "completed", "progress": 100}

event: artifact
data: {"partial": false, "content": {"final_result": "..."}}

3.2 协同使用

场景: Agent A 需要调用 Agent B 的能力，而 Agent B 内部使用 MCP 调用工具。

sequenceDiagram
    participant A as Agent A
    participant B as Agent B
    participant T as Tool (via MCP)
    
    A->>B: A2A: 创建 Task (code_review)
    B->>T: MCP: 调用静态分析工具
    T-->>B: MCP: 返回分析结果
    B->>T: MCP: 调用风格检查工具
    T-->>B: MCP: 返回风格评分
    B-->>A: A2A: 返回 Artifact (综合报告)

实现示例:

# Agent B 的服务端（支持 A2A）
from a2a_protocol import A2AServer, Task

server = A2AServer()

@server.skill("code_review")
def review_code(task: Task):
    code = task.input["content"]
    
    # 内部使用 MCP 调用工具
    from mcp import MCPClient
    mcp_client = MCPClient("static_analysis_tool")
    
    result = mcp_client.call_tool(
        tool_name="analyze",
        arguments={"code": code}
    )
    
    return {
        "artifacts": [{"type": "application/json", "content": result}]
    }

server.run(port=8080)

4. Agent 互操作实践

4.1 场景 1: 跨框架协作（AutoGen + CrewAI）

目标: AutoGen 的 AssistantAgent 调用 CrewAI 的 Researcher Agent。

方案: 使用 A2A 协议封装 CrewAI Agent，使其可以被 AutoGen 调用。

步骤:

  1. 封装 CrewAI Agent 为 A2A Server
# crewai_a2a_server.py
from crewai import Agent, Task, Crew
from a2a_protocol import A2AServer

# 创建 CrewAI Agent
researcher = Agent(role="Researcher", ...)
research_task = Task(description="Research AI trends", ...)
crew = Crew(agents=[researcher], tasks=[research_task])

# 封装为 A2A Server
server = A2AServer()

@server.skill("research")
def conduct_research(task: Task):
    topic = task.input["content"]
    result = crew.kickoff(inputs={"topic": topic})
    return {"artifacts": [{"content": result}]}

server.run(port=8080)
  1. AutoGen 调用 A2A Server
# autogen_client.py
import autogen
from a2a_protocol import A2AClient

# 创建 A2A Client
client = A2AClient(base_url="http://localhost:8080")

# 定义 AutoGen Agent
assistant = autogen.AssistantAgent(
    name="Assistant",
    llm_config={"config_list": [{"model": "gpt-4"}]}
)

# 自定义函数：调用 CrewAI Agent
def call_crewai_researcher(topic: str):
    task = client.create_task(skill_id="research", input=topic)
    result = client.wait_for_completion(task["task_id"])
    return result["artifacts"][0]["content"]

# 注册函数
assistant.register_function(
    name="conduct_research",
    function=call_crewai_researcher
)

# 使用
assistant.send("Research the latest AI trends and summarize.")

4.2 场景 2: 多厂商 Agent 协作（Google + Microsoft）

目标: Google 的 Vertex AI Agent 与 Microsoft 的 Copilot Studio Agent 协作。

方案: 双方都支持 A2A 协议，通过标准 HTTP API 通信。

前提:

  • Google Vertex AI Agent 已发布 A2A Endpoint
  • Microsoft Copilot Studio Agent 已发布 A2A Endpoint

示例代码:

# 调用 Google Vertex AI Agent
from a2a_protocol import A2AClient

google_agent = A2AClient(
    base_url="https://vertex-ai-agent.google.com/.well-known/agent.json"
)

# 获取 Agent Card
agent_card = google_agent.get_agent_card()
print(f"Google Agent skills: {agent_card['skills']}")

# 创建 Task
task = google_agent.create_task(
    skill_id="summarize_text",
    input="Please summarize this article..."
)

# 轮询结果
result = google_agent.wait_for_completion(task["task_id"])
print(result["artifacts"][0]["content"])

4.3 场景 3: 本地 Agent 与云 Agent 混合部署

目标: 本地部署的私有 Agent（处理敏感数据）与云端 Agent（使用强大模型）协作。

架构:

用户请求
  ↓
本地 Agent (数据脱敏)
  ↓
A2A 调用
  ↓
云端 Agent (数据处理)
  ↓
A2A 返回
  ↓
本地 Agent (结果聚合)
  ↓
返回用户

安全考虑:

  • 本地 Agent 在发送数据前脱敏（移除 PII）
  • 使用 HTTPS + Token 认证
  • 云端 Agent 不存储用户数据（无状态处理）

5. A2A 协议实现

5.1 Python 实现（a2a-python）

安装:

pip install a2a-protocol

创建 A2A Server:

from a2a_protocol import A2AServer, Task, Skill

server = A2AServer()

# 定义 Skill
@server.skill(
    id="translate",
    name="Translation",
    description="Translate text to target language",
    input_types=["text/plain"],
    output_types=["text/plain"]
)
def translate(task: Task):
    text = task.input["content"]
    target_lang = task.input.get("target_lang", "en")
    
    # 调用翻译模型
    result = translate_model.translate(text, target_lang)
    
    return {
        "artifacts": [
            {
                "type": "text/plain",
                "content": result
            }
        ]
    }

# 启动 Server
server.run(host="0.0.0.0", port=8080)

创建 A2A Client:

from a2a_protocol import A2AClient

client = A2AClient(base_url="http://localhost:8080")

# 获取 Agent Card
agent_card = client.get_agent_card()
print(agent_card)

# 创建 Task
task = client.create_task(
    skill_id="translate",
    input="Hello, world!",
    metadata={"target_lang": "zh"}
)

# 等待完成
result = client.wait_for_completion(task["task_id"], timeout=30)
print(result["artifacts"][0]["content"])  # 输出: 你好，世界！

6. A2A 协议的挑战与未来

6.1 当前挑战

挑战 描述 潜在解决方案
采用率 尚未成为行业标准，采用率较低 Google 推动，开源社区支持
安全性 Agent 之间的信任、认证、授权 OAuth 2.1 + PKCE（参考 MCP）
性能 HTTP 通信开销，不适合高频调用 gRPC、WebSocket 支持
状态管理 分布式状态同步复杂 使用 CRDT（Conflict-free Replicated Data Types）
版本兼容 协议版本升级的向后兼容 语义化版本控制 + 弃用策略

6.2 未来方向

1. A2A + MCP 深度融合

愿景: Agent 既可以调用工具（MCP），也可以调用其他 Agent（A2A），且接口统一。

示例:

# 统一的 Tool/Agent 调用接口
from agent_protocol import ToolClient, AgentClient

# 调用工具（MCP）
tool_client = ToolClient("static_analysis_tool")
tool_result = tool_client.call("analyze", {"code": "..."})

# 调用 Agent（A2A）
agent_client = AgentClient("code_reviewer_agent")
agent_result = agent_client.create_task("review", {"code": "..."})

# 统一返回格式
assert tool_result.format == agent_result.format  # True

2. Agent 服务发现

愿景: 类似 DNS 的 Agent 服务发现机制，Agent 可以动态发现其他 Agent。

示例:

from a2a_registry import AgentRegistry

registry = AgentRegistry()

# 注册 Agent
registry.register(
    agent_url="http://code-reviewer:8080",
    skills=["code_review", "security_audit"]
)

# 发现 Agent
agents = registry.discover(skill="code_review")
print(agents)  # ['http://code-reviewer:8080', 'http://code-expert:8080']

3. Agent 市场

愿景: 类似 AWS Marketplace 的 Agent 市场，开发者可以发布、销售、订阅 Agent。

示例:

from a2a_marketplace import Marketplace

marketplace = Marketplace()

# 浏览 Agent
agents = marketplace.search(
    category="code_review",
    rating_min=4.5,
    price_max=0.10  # 每次调用价格
)

# 订阅 Agent
subscription = marketplace.subscribe(
    agent_id="code_reviewer_v2",
    plan="pay_per_use"
)

# 使用 Agent
result = subscription.call("review", {"code": "..."})

4. 跨链 Agent（Blockchain + A2A）

愿景: 使用区块链记录 Agent 的信誉、交易、协作历史。

示例:

from a2a_blockchain import AgentReputation

reputation = AgentReputation()

# 查询 Agent 信誉
score = reputation.get_score(agent_url="http://code-reviewer:8080")
print(f"Agent reputation: {score}/100")  # 输出: 95/100

# 记录成功协作
reputation.record_success(
    agent_url="http://code-reviewer:8080",
    task_id="task_123"
)
