Multi-Agent 协作模式与实践

> **核心概念**: 多个专业化 Agent 通过协作完成复杂任务，发挥集体智能优势 > > **应用场景**: 复杂问题求解、大规模数据处理、多模态任务、分布式决策

核心概念: 多个专业化 Agent 通过协作完成复杂任务，发挥集体智能优势

应用场景: 复杂问题求解、大规模数据处理、多模态任务、分布式决策

1. 为什么需要 Multi-Agent 协作？

1.1 单 Agent 的局限性

  • 能力边界: 单个 Agent 的知识、工具、上下文窗口有限
  • 任务复杂度: 现代 AI 任务往往需要多种专业能力（编码、搜索、推理、创作）
  • 容错性: 单点故障风险，缺乏冗余机制
  • 可扩展性: 无法通过简单扩展提升处理能力

1.2 Multi-Agent 的优势

  • 专业化分工: 每个 Agent 专注于特定领域，提升整体质量
  • 并行处理: 多个 Agent 同时工作，加速任务完成
  • 鲁棒性: 某个 Agent 失败时，其他 Agent 可以接管或补偿
  • 可解释性: 协作过程可追溯，决策链清晰

2. 主流协作模式

2.1 中心化模式（Centralized）

架构: 一个 Coordinator Agent 负责任务分配和结果聚合，其他 Agent 作为 Worker。

graph TD
    User[用户] --> Coordinator[协调者 Agent]
    Coordinator --> Worker1[Worker Agent 1<br/>代码生成]
    Coordinator --> Worker2[Worker Agent 2<br/>代码审查]
    Coordinator --> Worker3[Worker Agent 3<br/>测试生成]
    Worker1 --> Coordinator
    Worker2 --> Coordinator
    Worker3 --> Coordinator
    Coordinator --> User

优点:

  • 控制逻辑简单，易于实现
  • 全局视角，优化资源分配
  • 适合任务分解明确的场景

缺点:

  • 协调者成为单点故障
  • 扩展性受限（协调者处理能力瓶颈）
  • Worker Agent 缺乏自主性

典型案例:

  • AutoGen 的 GroupChatManager
  • CrewAI 的 Process.sequential

2.2 去中心化模式（Decentralized）

架构: 所有 Agent 平等协作，通过消息传递或直接通信完成任务。

graph LR
    A[Agent A<br/>研究员] <--> B[Agent B<br/>工程师]
    B <--> C[Agent C<br/>测试员]
    C <--> A
    A <--> D[Agent D<br/>产品经理]
    B <--> D
    C <--> D

优点:

  • 无单点故障，鲁棒性强
  • 自主性强，适应动态环境
  • 易于水平扩展

缺点:

  • 协调复杂，可能出现冲突
  • 通信开销大
  • 难以保证全局最优

典型案例:

  • LangGraph 的 StateGraph 多节点协作
  • Agent Protocol 的 P2P 通信

2.3 层次化模式（Hierarchical）

架构: Agent 分为多个层次，高层负责战略决策，低层负责执行。

graph TD
    CEO[CEO Agent<br/>战略规划] --> Manager1[Manager Agent 1<br/>技术架构]
    CEO --> Manager2[Manager Agent 2<br/>产品设计]
    Manager1 --> Worker1[Worker Agent 1<br/>后端开发]
    Manager1 --> Worker2[Worker Agent 2<br/>前端开发]
    Manager2 --> Worker3[Worker Agent 3<br/>UI/UX设计]
    Manager2 --> Worker4[Worker Agent 4<br/>用户研究]

优点:

  • 结合中心化和去中心化的优势
  • 适合复杂、多层次的任务
  • 易于管理和调试

缺点:

  • 层次设计需要领域知识
  • 层间通信可能成为瓶颈
  • 低层 Agent 自主性受限

典型案例:

  • MetaGPT 的 SDE Team（Product Manager → Architect → Engineer → QA）
  • AutoGen 的 ConversableAgent 嵌套协作

2.4 混合模式（Hybrid）

架构: 根据任务特点动态切换协作模式。

示例:

  • 前期使用中心化模式进行任务规划
  • 中期使用去中心化模式进行并行执行
  • 后期使用中心化模式进行结果聚合

典型案例:

  • LangGraph 的 conditional_edges（根据状态动态路由）
  • CrewAI 的 Process.hierarchical（动态调整管理层级）

3.2 通信拓扑结构

拓扑 描述 适用场景
点对点（P2P） 两个 Agent 直接通信 简单任务、双边协作
广播（Broadcast） 一个 Agent 发送消息给所有 Agent 任务分发、状态同步
总线（Bus） 所有 Agent 连接到共享消息总线 松耦合系统、事件驱动
管道（Pipeline） Agent 串联成流水线 数据处理、任务链式执行

3.3 消息传递机制

同步通信（Synchronous）

  • 特点: 发送方阻塞等待接收方响应
  • 优点: 简单直观，易于调试
  • 缺点: 性能受限，容错性差
  • 示例: HTTP REST API、gRPC
# Agent A 同步调用 Agent B
response = requests.post(
    "http://agent-b:8000/invoke",
    json={"message": "Analyze this data"}
)
result = response.json()["output"]

异步通信（Asynchronous）

  • 特点: 发送方不阻塞，通过回调或轮询获取结果
  • 优点: 高性能，松耦合
  • 缺点: 复杂性高，需要状态管理
  • 示例: Message Queue（RabbitMQ、Kafka）、Webhook
# Agent A 异步调用 Agent B
task_id = requests.post(
    "http://agent-b:8000/async_invoke",
    json={"message": "Analyze this data"}
).json()["task_id"]

# 轮询结果
while True:
    status = requests.get(f"http://agent-b:8000/task/{task_id}").json()
    if status["state"] == "completed":
        result = status["result"]
        break
    time.sleep(1)

4. 任务分解与结果聚合

4.1 任务分解策略

水平分解（Horizontal Decomposition）

将任务拆分为多个独立的子任务，并行执行。

示例: 代码审查任务可以分解为：

  • 子任务 1: 检查代码风格
  • 子任务 2: 检查逻辑错误
  • 子任务 3: 检查性能问题
  • 子任务 4: 检查安全漏洞
from concurrent.futures import ThreadPoolExecutor

def review_code_parallel(code):
    with ThreadPoolExecutor(max_workers=4) as executor:
        futures = [
            executor.submit(check_style, code),
            executor.submit(check_logic, code),
            executor.submit(check_performance, code),
            executor.submit(check_security, code)
        ]
        results = [f.result() for f in futures]
    return aggregate_results(results)

垂直分解（Vertical Decomposition）

将任务按处理阶段拆分，串行执行。

示例: 软件开发任务可以分解为：

  • 阶段 1: 需求分析
  • 阶段 2: 架构设计
  • 阶段 3: 代码实现
  • 阶段 4: 测试验证
from langgraph.graph import StateGraph

workflow = StateGraph(state_schema=dict)

workflow.add_node("requirements", requirements_agent)
workflow.add_node("architecture", architecture_agent)
workflow.add_node("implementation", implementation_agent)
workflow.add_node("testing", testing_agent)

workflow.add_edge("requirements", "architecture")
workflow.add_edge("architecture", "implementation")
workflow.add_edge("implementation", "testing")

app = workflow.compile()
result = app.invoke({"input": "Build a REST API..."})

4.2 结果聚合策略

投票机制（Voting）

多个 Agent 独立给出答案，通过投票选择最终结果。

示例: 3 个 Agent 分别判断代码是否有 bug：

  • Agent A: 有 bug
  • Agent B: 无 bug
  • Agent C: 有 bug

最终结果: 有 bug（2:1 投票）

def vote(results):
    from collections import Counter
    counter = Counter(results)
    return counter.most_common(1)[0][0]

加权聚合（Weighted Aggregation）

根据 Agent 的历史表现或专业度分配权重。

示例:

  • Agent A（专家）: 权重 0.6
  • Agent B（中级）: 权重 0.3
  • Agent C（初级）: 权重 0.1
def weighted_aggregate(results, weights):
    return sum(r * w for r, w in zip(results, weights))

迭代优化（Iterative Refinement）

多个 Agent 轮流改进结果，直到满足终止条件。

示例: 写作任务：

  1. Agent A 写初稿
  2. Agent B 提出修改建议
  3. Agent A 根据建议修改
  4. 重复 2-3 直到满意
def iterative_refinement(draft, max_iterations=3):
    current = draft
    for i in range(max_iterations):
        feedback = reviewer_agent.invoke({"draft": current})
        if feedback["satisfied"]:
            break
        current = writer_agent.invoke({
            "draft": current,
            "feedback": feedback["suggestions"]
        })
    return current

5. 冲突解决与一致性保证

5.1 冲突类型

冲突类型 描述 示例
目标冲突 Agent 之间的目标不一致 Agent A 追求速度，Agent B 追求质量
资源冲突 Agent 竞争有限资源 两个 Agent 同时访问同一个文件
结果冲突 Agent 给出矛盾的结果 Agent A 说"安全"，Agent B 说"有漏洞"
时序冲突 Agent 执行顺序导致问题 Agent A 删除了 Agent B 需要的数据

5.2 冲突解决机制

权威决策（Authority-Based）

指定一个权威 Agent（如协调者），由其做出最终决策。

def resolve_conflict(results):
    # 协调者 Agent 评估各结果
    scores = [coordinator.evaluate(r) for r in results]
    return results[scores.index(max(scores))]

谈判协商（Negotiation）

Agent 之间通过谈判达成一致。

示例: 合同谈判场景

  1. Agent A 提出初始报价
  2. Agent B 提出还价
  3. 双方轮流调整，直到达成协议或超时
def negotiate(proposal_a, proposal_b, max_rounds=5):
    for round in range(max_rounds):
        if acceptable(proposal_a, proposal_b):
            return (proposal_a, proposal_b)
        proposal_b = adjust_proposal(proposal_b, proposal_a)
        proposal_a = adjust_proposal(proposal_a, proposal_b)
    return fallback_solution()

投票共识（Consensus Voting）

Agent 通过多轮投票逐步收敛到共识。

def consensus_vote(proposals, threshold=0.8):
    while True:
        votes = [vote(proposal) for proposal in proposals]
        consensus = max(votes, key=votes.count)
        if votes.count(consensus) / len(votes) >= threshold:
            return consensus
        # 淘汰支持最少的提案
        proposals = eliminate_weak_proposals(proposals, votes)

分布式锁（Distributed Lock）

防止多个 Agent 同时修改共享资源。

import redis

版本控制（Version Control）

使用 Git 类似的机制管理 Agent 生成的内容。

事务机制（Transaction）

保证一系列操作要么全部成功，要么全部失败。

from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

engine = create_engine("sqlite:///agent_collab.db")
Session = sessionmaker(bind=engine)

def collaborative_task():
    session = Session()

        # Agent A 的操作
        session.add(AgentAResult(...))
        # Agent B 的操作
        session.add(AgentBResult(...))
        # 全部成功才提交
        session.commit()
    except:

        session.rollback()
        raise

6.2 CrewAI 角色化协作

from crewai import Agent, Task, Crew, Process

# 定义 Agent（角色）
researcher = Agent(
    role="Senior Researcher",
    goal="Uncover groundbreaking technologies",
    backstory="You are a seasoned researcher with a knack for uncovering the next big thing.",
    verbose=True,
    allow_delegation=False
)

writer = Agent(
    role="Writer",
    goal="Craft compelling content on technology advancements",
    backstory="You are a creative writer with a passion for technology.",
    verbose=True,
    allow_delegation=False
)

# 定义 Task
research_task = Task(
    description="Research the latest advancements in AI in 2026.",
    expected_output="A detailed report on AI advancements.",
    agent=researcher
)

write_task = Task(
    description="Write a blog post based on the research report.",
    expected_output="A engaging blog post.",
    agent=writer
)

# 组建 Crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    process=Process.sequential,  # 顺序执行
    verbose=True
)

# 启动协作
result = crew.kickoff()
print(result)

执行流程:

  1. Researcher 执行 research_task，输出研究报告
  2. Writer 接收研究报告，执行 write_task，输出博客文章

6.3 LangGraph 状态机协作

from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, END

# 定义状态
class AgentState(TypedDict):
    messages: list
    next: str

# 定义 Agent
llm = ChatOpenAI(model="gpt-4")

def researcher_node(state: AgentState):
    messages = state["messages"]
    response = llm.invoke(messages)
    return {"messages": messages + [response], "next": "writer"}

def writer_node(state: AgentState):
    messages = state["messages"]
    response = llm.invoke(messages)
    return {"messages": messages + [response], "next": "END"}

# 构建状态图
workflow = StateGraph(AgentState)

workflow.add_node("researcher", researcher_node)
workflow.add_node("writer", writer_node)

workflow.set_entry_point("researcher")
workflow.add_edge("researcher", "writer")
workflow.add_edge("writer", END)

# 编译并执行
app = workflow.compile()
result = app.invoke({
    "messages": [("user", "Write about Multi-Agent systems.")],
    "next": "researcher"
})

执行流程:

  1. 初始状态进入 researcher 节点
  2. Researcher 生成研究内容
  3. 状态转移到 writer 节点
  4. Writer 基于研究内容生成文章
  5. 状态转移到 END，结束

7.3 性能优化

from concurrent.futures import ThreadPoolExecutor

def parallel_execution(agents, task):
    with ThreadPoolExecutor(max_workers=len(agents)) as executor:
        futures = [executor.submit(agent.invoke, task) for agent in agents]
        results = [f.result() for f in futures]
    return results

7.4 安全与隐私

访问控制

from functools import wraps

def require_auth(f):
    @wraps(f)
    def wrapper(*args, **kwargs):
        if not kwargs.get("auth_token"):
            raise PermissionError("Authentication required")
        return f(*args, **kwargs)
    return wrapper

@require_auth
def sensitive_operation(agent, task, auth_token):
    return agent.invoke(task)

数据脱敏

import re

def sanitize_data(text):
    # 移除邮箱
    text = re.sub(r'\b[\w.-]+@[\w.-]+\.\w+\b', '[EMAIL]', text)
    # 移除手机号
    text = re.sub(r'\b\d{11}\b', '[PHONE]', text)
    return text

8. 评估指标

指标 描述 计算方法
任务完成率 成功完成的任务占比 成功任务数 / 总任务数
协作效率 完成任务所需的平均 Agent 数 总 Agent 调用次数 / 完成任务数
通信开销 Agent 之间传递的消息总数 统计消息传递次数
冲突率 发生冲突的任务占比 冲突任务数 / 总任务数
容错性 Agent 失败时的恢复能力 模拟 Agent 失败，测量成功率下降

9. 未来方向

9.1 自适应协作

Agent 根据任务特点动态调整协作模式（中心化 ↔ 去中心化）。

9.2 元学习协作

Agent 通过学习历史协作经验，优化未来的协作策略。

9.3 跨平台协作

不同框架（AutoGen、CrewAI、LangGraph）的 Agent 之间实现互操作。

9.4 人机协作

人类作为特殊的 Agent 参与协作，提供监督、反馈、决策。
