对话驱动的企业工作台：操作卡片模板

2026-06-09前端交互企业

在 Agent 对话中直接渲染可交互的表单卡片，用户说话就能完成请假、报销、报修等操作。这套"操作卡片"模板是对话式企业工作台的核心交互范式。

已落库的卡片模板

卡片类型触发示例提交接口
leave_form"我要请假"POST /api/leaves
expense_form"我要报销"POST /api/expenses
ticket_form"电脑坏了"POST /api/tickets
attendance_punch"打卡"POST /api/attendance/punch

实现机制

  1. Orchestrator 的 _detect_action_card() 识别用户意图 → 返回卡片类型
  2. 前端 renderActionCard(card) 根据 card.type 渲染对应表单
  3. bindCardEvents() 绑定提交按钮 → 调用对应后端 API
  4. 数据写入数据库 → 进入审批/工单流程

设计理念

用户不需要学习菜单和表单，说话即可完成所有日常工作事务。Agent 识别意图后渲染操作卡片，用户一步完成提交。

当前支持 4 种落库模板，另有 2 种（加班申请、出差申请）已有前端雏形待完善后端链路。
