Skills 层设计：Agent 应用中的业务抽象

2026-06-04架构Skills设计模式

在企业多专家 Agent 系统中，我们引入了一个关键的抽象层——Skills 层，解决了 Agent 框架中"业务逻辑散落"的痛点。

痛点

Skills 层的定位

用户界面 (CLI / Web / API)
         │
    ┌────▼────┐
    │  Agent  │  ← ReAct 循环
    └────┬────┘
         │ 调用
    ┌────▼────┐
    │  Skills  │  ← 业务能力层
    │ 审批引擎 │
    │ 工单状态机│
    │ 请假流程 │
    └─────────┘

Skills 层将多步业务流转（审批、工单、请假）封装为独立的能力模块，Agent 通过 ReAct 调用它们，让业务逻辑可复用、可测试、可独立迭代。

层示例问题
tools/create_ticket只有原子操作，无状态机、无生命周期管理

agents/base_agent.runReAct 只适合单步推理，不适合多步事务
