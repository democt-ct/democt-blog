代码优化实践：跨域合并回答与上下文隔离

2026-06-05优化重构代码质量

企业多专家 Agent 系统在一次全量代码分析后，识别出几个关键优化点。

1. 跨域合并回答综合摘要

问题：HR Agent 和财务 Agent 并行执行后，回答直接字符串拼接，用户体验割裂。

方案：追加一次 LLM 调用做综合摘要，输出连贯的统一回答。同时保留原始拼接结果在 raw_answers 中供 trace 查看。

2. 用户上下文移出 user 消息

问题：用户身份通过字符串拼接注入到 user 消息内容，污染对话历史和 token 计数。

方案：用户上下文作为独立 system 消息注入。AgentRequest 新增 user_context 字段，不在 query 中拼接。base_agent 的 react_loop 中追加第二条 system 消息携带上下文。

3. JWT 加入过期时间

问题：Token 永不过期，泄露即永久沦陷。

方案：Access Token 8h + Refresh Token 7d，支持续期。新增 refresh_tokens 表，前端拦截 401 自动续期。

优化原则：每次改动前后都有明确的验收标准，不引入新的技术债。
