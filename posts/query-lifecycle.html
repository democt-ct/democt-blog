患者 Agent 问答的完整生命周期

2026-05-20医疗架构完整流程

用户在患者 Agent 发一条消息，背后发生了什么？从输入到输出共 7 个阶段。

Step 1 → 身份解析

Auth Token 校验 → 自然语言身份提取（姓名/出生年/手机号）→ Session ID 分配 → 结果：确定 patient_id + hospital_id

Step 2 → 图片分析（可选）

如果请求包含图片 → 调用视觉 LLM 提取图片摘要（报告文字/药品包装/检查结果/部位照片）

Step 3 → 短期工作记忆更新

更新当前会话的意图、主题、目标。维护最近 5 轮消息、活跃实体（药物/症状/检查/指标）、风险信号。

Step 4 → 长期记忆上下文构建

从数据库中加载四类长期信息：患者主档、业务画像（诊断/风险/护理）、对话画像（沟通偏好/聚焦话题）、关键事件、病历和就诊记录。

Step 5 → 知识检索（混合 RAG）

向量召回（语义相似） + 关键词补召回（精确命中） + 元数据过滤 + 排序融合。

Step 6 → LLM 推理组装

将短期记忆 + 长期记忆 + 知识检索结果 → 组装为完整 Prompt → 调用 LLM 生成回答。LLM 可选用 MCP 工具获取更精确的实时数据。

Step 7 → 输出与记忆持久化

回答返回给用户 → 异步更新长期摘要记忆（用户画像/偏好/事件）→ 如果是图片回答，可触发 TTS 语音播报。

每个阶段都有独立的调试入口，方便追踪和定位问题。
